## Claim

**Pre-launch (days -7 to -1):**
- README with demo GIF, badges, install instructions
- CI green on macOS 13/14/15 + Python 3.11-3.13
- Published to PyPI (`pip install apple-a-day` works)
- Homebrew tap created (`brew tap eidos-agi/tap && brew install apple-a-day`)
- 3-5 issues tagged `good first issue`
- CHANGELOG.md, SECURITY.md, CODE_OF_CONDUCT.md in place
- Seed 10-15 stars from personal network (social proof)

**Launch week (days 1-7):**
- Day 1 (Tue/Wed): Show HN + Product Hunt simultaneously
- Day 2: r/commandline
- Day 3: r/macOS ("free alternative" angle)
- Day 4: r/Python (code quality angle)
- Day 5: Dev.to technical writeup
- All week: respond to every issue/comment within 24 hours
- Target: 50-100 stars

**Month 1 (weeks 2-4):**
- Ship v0.2.0 (thermal monitoring, network diagnostics)
- Submit to 4 awesome-lists (one PR per week)
- Add `aad checkup --watch` for continuous monitoring
- Create GitHub Discussions
- Target: 200-500 stars

**Quarter 1 (months 2-3):**
- Ship v0.3.0 with `aad history` (health over time, SQLite)
- Ship v0.4.0 with `aad fix` (actionable remediation)
- Submit to homebrew-core
- Docs site via mkdocs-material on GitHub Pages
- Conference lightning talk submissions (PyCon, MacAdmins)
- Target: 500-1000 stars
