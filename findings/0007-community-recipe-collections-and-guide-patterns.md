## Claim

Several community resources aggregate Mac diagnostic workflows into step-by-step recipes that apple-a-day can codify:

1. **Setapp diagnostic guide** — aggregates best practices around Apple Diagnostics, Disk Utility, Activity Monitor, Console.app, and network tools.
2. **Avast Mac performance guide** — covers benchmarking, storage cleanup, and monitoring patterns.
3. **WikiHow / eclecticlight diagnostics patterns** — practical step-by-step recipes for using Apple Diagnostics/AHT and reading results.
4. **Recovery Selector (Twocanoes)** — open-source app to select next boot into Recovery/Diagnostics. Relevant pattern for "one-click" health flows.

The value here isn't the content itself but the user workflows they document — these are the "jobs to be done" that apple-a-day should automate.
